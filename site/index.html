核心特性

你需要的，都在这里

从系统代理到 HTTPS 抓包，从规则路由到配置管理，WhistleBox 将 Whistle 的全部能力封装在一个精致的桌面应用中。

系统级代理

全局 / 规则 / 直连三种模式，一键切换。托盘图标实时反映当前状态，右键菜单快捷操作。

规则路由

PAC 驱动的域名级精细代理控制，支持通配符与精确匹配。按需代理，不干扰其他流量。

内嵌 Whistle

自带 Node.js 运行时和 Whistle 环境，开箱即用。不依赖系统 Node 版本，不污染全局环境。

自动认证

内嵌界面自动注入 Token 认证，无需手动输入账号密码。CSPRNG 一次性 Token + Origin 校验双保险。

进程保活

内嵌 Whistle 自动启动、周期健康检查、连续失败自动重启。异常恢复静默完成，无感知。

外部支持

不仅支持内嵌模式，也可连接外部 Whistle 实例。自动探测存活状态，灵活适配不同工作流。

配置管理

多 Profile 切换，支持 Whistle 规则与应用配置的导入导出。原子写入防损坏，损坏自动恢复。

HTTPS 抓包

一键安装/卸载根证书，开启 HTTPS 拦截。证书状态与 Whistle 配置自动同步。

系统架构

四层分离，各司其职

React 负责界面渲染，Tauri IPC 桥接前后端，Rust 处理系统级操作，Node.js Sidecar 承载 Whistle 实例。

快速开始

三步启动

从克隆仓库到看见界面，只需要几条命令。

前置条件

Rust 1.70+Windows 需安装 MSVC 工具链
Node.js 18+仅开发环境需要
系统依赖Linux: libwebkit2gtk-4.1-dev, libappindicator3-dev

跨平台支持

■ Windows ● macOS ▲ Linux
Terminal
# 克隆仓库
$git clone https://github.com/fu9zhou/whistle-box.git
$cd whistle-box
# 安装依赖 + 准备 Node.js sidecar
$npm run setup
# 启动开发模式
$npm run start
# 构建安装包（Windows）
$npm run package
安全模型

安全不是附加项，是设计原则

从网络边界到进程管理，每一层都经过安全审视。

网络边界

  • Whistle 仅允许回环/私网地址
  • Auth proxy / PAC 服务绑定 127.0.0.1
  • 退出时自动清除系统代理

认证授权

  • CSPRNG 一次性 Token 鉴权
  • Referer / Origin 双校验
  • local_auth_bypass 默认关闭

进程管理

  • Fail-closed 端口占用识别策略
  • 配置原子写入防损坏
  • Node.js sidecar SHA-256 校验
  • npm ci + lockfile 可复现构建
